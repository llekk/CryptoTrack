# Описание исходного кода CryptoTrack

В этой папке находятся исходные файлы приложения **CryptoTrack**.  
Приложение написано на C++ с использованием Qt Widgets и Qt Network.

---

## Состав файлов

```text
project/
├── project.pro
├── main.cpp
├── ai_project.h
├── ai_project.cpp
└── ai_project.ui
```

---

## project.pro

Файл конфигурации qmake.

В нём подключаются основные Qt-модули:

```pro
QT += core gui widgets network
```

Используемые модули:

- `core` — базовые классы Qt;
- `gui` — графические элементы;
- `widgets` — интерфейс на QWidget;
- `network` — HTTP-запросы к CoinGecko API.

Также в файле указаны исходники, заголовочные файлы и форма интерфейса.

---

## main.cpp

Точка входа в приложение.

Основные действия:

1. Создаётся объект `QApplication`.
2. Создаётся главное окно `ai_project`.
3. Окно отображается через `show()`.
4. Запускается главный цикл приложения через `app.exec()`.

Файл минимальный, вся основная логика находится в `ai_project.h` и `ai_project.cpp`.

## Основные структуры данных

### CandleUiData

```cpp
struct CandleUiData
{
    double open;
    double high;
    double low;
    double close;
};
```

Используется для хранения одной свечи графика.

Поля:

- `open` — цена открытия;
- `high` — максимальная цена;
- `low` — минимальная цена;
- `close` — цена закрытия.

---

### CoinUiData

```cpp
struct CoinUiData
{
    QString id;
    QString symbol;
    QString name;

    double price;
    double changePercent;
    double volume;

    QVector<double> history;
    QVector<double> volumes;
    QVector<double> rsi;
    QVector<CandleUiData> candles;
};
```

Хранит данные по одной криптовалюте.

Используется для:

- отображения цены;
- построения графиков;
- расчёта RSI;
- построения свечей;
- работы конвертора.

---

### PriceAlertUiData

```cpp
struct PriceAlertUiData
{
    QString coinId;
    QString coinSymbol;

    double targetPrice;
    bool aboveCondition;
    bool active;
};
```

Хранит одно ценовое оповещение.

Поля:

- `coinId` — внутренний id монеты CoinGecko;
- `coinSymbol` — тикер монеты;
- `targetPrice` — целевая цена;
- `aboveCondition` — условие “цена выше” или “цена ниже”;
- `active` — активно ли оповещение.

## Работа с CoinGecko API

### fetchMarketData()

Отправляет запрос на получение текущих цен криптовалют.

Используется endpoint CoinGecko:

```text
/api/v3/coins/markets
```

Метод получает:

- текущую цену;
- изменение за 24 часа;
- объём торгов.

Также в методе есть защита от слишком частых запросов.

---

### handleMarketDataReply()

Обрабатывает ответ от CoinGecko.

Если запрос успешный:

- обновляет цены;
- обновляет левую панель;
- обновляет центральную карточку;
- проверяет оповещения.

Если запрос неудачный:

- обрабатывает ошибку;
- при HTTP-коде `429` сообщает о лимите запросов;
- приложение не завершает работу аварийно.

---

### updateCoinFromJson()

Обновляет данные конкретной монеты на основе JSON-ответа API.

Обновляются:

- цена;
- изменение за 24 часа;
- объём торгов;
- последние значения истории;
- последняя свеча;
- RSI.

---

### fetchHistoryData()

Загружает исторические данные выбранной монеты.

История загружается при:

- выборе другой монеты;
- переключении периода.

---

### handleHistoryDataReply()

Обрабатывает исторические данные.

Из ответа API берутся:

- цены;
- объёмы торгов.

После этого данные преобразуются в формат, удобный для графиков.

---

## Построение графиков

### reduceToClosingValues()

Уменьшает количество точек истории до нужного количества.

Например:

- 24 часа → 24 точки;
- 7 дней → 42 точки;
- 30 дней → 30 точек;
- 90 дней → 90 точек.

---

### buildCandlesFromPrices()

Строит свечи на основе исторических цен.

Для каждого интервала рассчитываются:

- `open`;
- `high`;
- `low`;
- `close`.

---

### closeValuesFromCandles()

Получает цены закрытия из массива свечей.

Эти значения используются для линейного графика и расчёта RSI.

---

## RSI

### calculateRsiValues()

Рассчитывает RSI по ценам закрытия.

Используется период 14 точек.

RSI показывает силу движения цены:

- выше 70 — зона перекупленности;
- ниже 30 — зона перепроданности.

---

## Уровни поддержки и сопротивления

### calculateSupportLevels()

Ищет уровни поддержки по локальным минимумам цены.

Поддержка — это область, где цена ранее разворачивалась вверх.

---

### calculateResistanceLevels()

Ищет уровни сопротивления по локальным максимумам цены.

Сопротивление — это область, где цена ранее разворачивалась вниз.

---

## Оповещения

### addAlert()

Добавляет новое ценовое оповещение.

Пользователь выбирает:

- монету;
- условие;
- целевую цену.

После добавления оповещение сохраняется в файл.

---

### checkPriceAlerts()

Проверяет активные оповещения.

Если условие выполнено:

- появляется окно уведомления;
- оповещение получает статус `сработало`;
- состояние сохраняется в файл.

---

### deleteSelectedAlert()

Удаляет выбранное оповещение из списка.

Перед удалением выводится подтверждение.

---

### clearTriggeredAlerts()

Удаляет только сработавшие оповещения.

Активные оповещения не удаляются.

---

### refreshAlertsList()

Обновляет список оповещений на правой панели.

Активные и сработавшие оповещения отображаются разным цветом.

## Автообновление

Автообновление реализовано через `QTimer`.

Интервал обновления:

```text
120 секунд
```

При срабатывании таймера вызывается:

```cpp
fetchMarketData();
```

После обновления цен автоматически проверяются оповещения.

---

## Конвертор криптовалют

### updateConverter()

Рассчитывает количество одной криптовалюты через стоимость другой.

Формула:

```text
result = amount * priceFrom / priceTo
```

Все цены считаются относительно USD.

---

## Обработка ошибок

В приложении обработаны основные ошибки:

- нет интернета;
- CoinGecko не отвечает;
- API вернул неожиданный JSON;
- превышен лимит запросов;
- VPN мешает подключению;
- история цены не загружена.

При ошибке приложение не закрывается, а продолжает работу на последних доступных данных.

---

## Принцип работы приложения

Общий сценарий работы:

1. При запуске создаются демонстрационные данные.
2. Загружаются сохранённые оповещения и настройки.
3. Приложение запрашивает текущие цены CoinGecko.
4. Пользователь выбирает монету.
5. Загружается история выбранной монеты.
6. Строятся графики, RSI, свечи и уровни.
7. Пользователь может добавить оповещение или выполнить конвертацию.
8. При обновлении цены проверяются активные оповещения.
9. При закрытии сохраняются настройки и оповещения.
